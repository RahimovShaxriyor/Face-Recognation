# SmartFace

Профессиональное desktop-приложение для распознавания лиц на базе **JavaFX** и **OpenCV**. Проект построен в модульном стиле: камера, распознавание, хранение профилей пользователей, интерфейс и визуальные overlay-элементы разделены по отдельным классам.

## О проекте

**SmartFace** — это приложение для работы с камерой и распознаванием лиц в реальном времени. Система умеет:

- подключаться к веб-камере;
- обнаруживать лицо в видеопотоке;
- отображать визуальные рамки и HUD-элементы поверх лица;
- сохранять пользователей;
- хранить профили пользователей;
- подготавливать основу для последующего обучения и распознавания лиц;
- использовать более профессиональный интерфейс вместо одного перегруженного класса.

Проект создавался как более структурированная и расширяемая версия обычного face-recognition приложения.

## Основные возможности

- Захват изображения с камеры в реальном времени
- Детекция лица через OpenCV
- Визуальные рамки вокруг обнаруженного лица
- Overlay-режимы для более профессионального отображения
- Сохранение профиля пользователя
- Отдельный список пользователей
- Подготовленная архитектура для расширения логики распознавания
- Разделение UI, сервисов, моделей и контроллеров

## Назначение классов

### `SmartFaceApplication`
Главная точка входа в приложение. Запускает JavaFX и инициализирует основной интерфейс.

### `MainController`
Связывает интерфейс с сервисами. Управляет логикой запуска камеры, обработкой кадров, сохранением пользователей и работой с UI.

### `OverlayMode`
Перечисление режимов отображения overlay-элементов на экране.

### `FaceOverlay`
Модель для хранения информации о найденном лице: координаты, размеры, подписи, уровень уверенности и визуальные параметры.

### `UserProfile`
Модель пользователя. Может содержать имя, идентификатор, путь к сохранённому изображению и другие метаданные.

### `CameraService`
Отвечает за подключение к камере, получение видеокадров и управление ресурсами устройства.

### `RecognitionService`
Содержит логику детекции и будущего распознавания лиц через OpenCV.

### `StorageService`
Сохраняет данные пользователей, изображения и локальные метаданные.

### `HudRenderer`
Рисует профессиональные overlay-элементы: рамки, углы, подписи, индикаторы и прочие графические детали.

### `MainView`
Главное окно приложения. Формирует визуальную часть интерфейса.

### `FxImageUtils`
Вспомогательный класс для конвертации изображений OpenCV в формат JavaFX.

## Как запустить проект

### 1. Убедитесь, что установлены

- Java 17 или новее
- Maven
- JavaFX
- OpenCV

Проверка:

```bash
java -version
mvn -version
```

### 2. Клонируйте проект

```bash
git clone <URL_ВАШЕГО_РЕПОЗИТОРИЯ>
cd face-Recognation
```

### 3. Создайте структуру каталогов

Если структура ещё не создана:

```bash
mkdir -p src/main/java/org/controller
mkdir -p src/main/java/org/model
mkdir -p src/main/java/org/service
mkdir -p src/main/java/org/ui
mkdir -p src/main/java/org/util
```

### 4. Установите OpenCV

Для macOS через Homebrew:

```bash
brew install opencv
```

Если проект использует модуль `opencv_face`, важно, чтобы в установленной версии OpenCV присутствовал пакет `org.opencv.face`.

### 5. Проверьте зависимости Maven

Если при сборке появляются ошибки вида:

- `package org.opencv.face does not exist`
- `cannot find symbol class LBPHFaceRecognizer`

это означает, что в проекте не подключён модуль OpenCV Face или используется неподходящая Java-обёртка.

### 6. Сборка проекта

```bash
mvn clean compile
```

### 7. Запуск приложения

Если используется JavaFX Maven Plugin:

```bash
mvn javafx:run
```

## Типичные ошибки и решения

### Ошибка: `zsh: no such file or directory`
Причина: путь к `.java` файлу был введён в терминал как команда.

Неправильно:

```bash
src/main/java/org/SmartFaceApplication.java
```

Правильно создавать файл так:

```bash
touch src/main/java/org/SmartFaceApplication.java
```

### Ошибка: `package org.opencv.face does not exist`
Причина: OpenCV установлен без нужного Java-модуля face, либо Maven не видит корректную библиотеку.

Что проверить:

- подключён ли OpenCV Java binding;
- совпадает ли версия `opencv` и `opencv_java`;
- доступен ли модуль `org.opencv.face`;
- правильно ли загружается native library.

### Ошибка: `cannot find symbol LBPHFaceRecognizer`
Причина: класс распознавания не найден из-за отсутствия `opencv_contrib` или неподходящей сборки OpenCV.

## Рекомендуемое улучшение структуры пакетов

Сейчас проект может работать в пакете `org`, но для более чистой архитектуры лучше использовать именованное пространство, например:

```text
org.smartface
org.smartface.controller
org.smartface.model
org.smartface.service
org.smartface.ui
org.smartface.util
```

Это улучшает читаемость, масштабируемость и поддержку проекта.
